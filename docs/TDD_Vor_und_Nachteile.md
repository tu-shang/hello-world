# Vor- und Nachteile von TDD

## Vorteile
- Fehler werden früh erkannt.
- Der Code wird testbarer und oft sauberer aufgebaut.
- Anforderungen werden genauer durchdacht.
- Refactoring ist sicherer, weil Tests sofort Rückmeldung geben.
- Die Tests dienen gleichzeitig als technische Dokumentation.

## Nachteile
- Am Anfang braucht TDD mehr Zeit und Disziplin.
- Für Einsteiger ist die Denkweise zuerst ungewohnt.
- Schlechte oder unklare Tests können die Entwicklung bremsen.
- Nicht jede kleine Änderung lässt sich gleich sinnvoll als Test formulieren.
- Bei sehr kleinen Projekten wirkt der Aufwand manchmal hoch.

## Fazit
TDD lohnt sich besonders bei Projekten, die wachsen oder länger gepflegt
werden. Es verbessert die Qualität und Nachvollziehbarkeit des Codes,
verlangt aber sauberes Arbeiten und Übung.
